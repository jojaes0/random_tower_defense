> 이 문서 하나만 읽으면 다른 PC/다른 작업자(또는 다른 세션의 AI)가 곧바로 이어서 작업할 수 있도록
> 이번 세션의 모든 결정·진행 상황·미결 사항을 정리한 문서. (작성 시점 기준 최신)

## 0. 한 줄 요약
스타크래프트2 아케이드 **「랜덤 타워 디펜스」** 를 **Vue 3 + TypeScript(Vite)** 로 재구현. 가로 레이아웃,
Canvas 게임 필드 + Vue 반응형 HUD. 나무위키 구조 + 사용자 실측 데이터를 기준으로 밸런싱.

## 1. 다른 PC에서 시작하기
```bash
# 폴더 전체 복사(또는 git clone) 후
npm install
npm run dev      # http://localhost:5173 에서 플레이 (자동으로 브라우저 열림)
npx vue-tsc --noEmit   # 타입체크
```
- ⚠️ **프로덕션 빌드 주의**: `npm run build`는 **Node 24 + Windows** 조합에서 rollup 네이티브 단계
  크래시(`0xC0000409`)가 납니다(게임 코드 무관, hello-world도 동일). 빌드가 필요하면 **Node 20/22 LTS**
  에서 하세요. 개발/플레이는 `npm run dev`(esbuild)로 영향 없음. 자세한 건 [README.md](README.md).

## 4. 구현 완료 기능 (이번 세션 누적)
- **핵심 루프**: 50라운드·10마다 보스·50클리어 승리. 종족 미선택, **일반 타워만 랜덤 건설(100)**,
  **같은 종류 2개 합성 → 상위 등급**(일반→희귀→영웅→전설→신). 판매 반값.
- **합성 모드**: 도크 토글 버튼(일반 타워/가스와 동일 스타일). 켜고 타워 클릭 시 짝이 있으면 즉시 합성.
- **경제**: 시작 미네랄 300, 라운드 클리어 +300, 처치당 미네랄 없음(전부 50/100 단위).
- **가스**: 미네랄 100 → 가스 도박(20~120, 10단위).
- **3종족 업그레이드**: 시작 20가스, 레벨당 +2가스, **공격력 +12.5%/레벨(단리)**, 최대 100업.
- **테라진**: 보스마다 +1, 750킬 +2, 3종족 신 +1 / 사용: 미네랄400·가스300·영웅 선택권(◆1+100원).
- **퀘스트 9종**(수집5 + 750킬 + 시간 7/12/18분), **달성 토스트** 안내.
- **개인 미션 3단계**: 게/암흑광전사/대군주, **5분 쿨다운**, 체력=나무위키 절대값(3000/30000/175000)×난이도,
  **roundBound 분리**(라운드 중 소환=라운드 종료 막음 / 대기 중 소환=라운드와 별개로 진행).
- **시그니처 스킬**: 사도 분신(2%), 불곰 감속, 수호군주·악령·고위기사 다단/스플래시.
- **난이도 4종**(쉬움300%/보통400%/어려움500%/지옥500%목숨5+보너스), **무한모드**(50클리어 후 테라진222 + 51R~).
- **시간/진행**: 게임 클럭, **배속 1/2/3x·일시정지**(배속이 카운트·시간경과에도 적용),
  **첫 라운드만 수동, 이후 20초 카운트다운 후 자동 시작**(미리보기 칩에 표시).
- **맵**: 사용자 스프레드시트 12×12 미로(방=설치칸/복도=경로). 경로는 **벽에 막힐 때만 꺾이는 serpentine**.
  그리드 셀 48px 스냅 배치(셀당 1타워), 타워는 **아이콘+이름+종족테두리**로 표시.
- **사거리**: 1유닛 = 0.5칸(`RANGE_SCALE=24`). 전설·신은 나무위키 실측 사거리.
- **체력**: `roundHp.ts` 50칸 실측 테이블(지옥 기준), 난이도 `×hpMult/5`. 보스 라운드는 테이블값=보스HP.
- **이동속도**: 모든 몹 4.5(=108px/s) 공통.
- **패배 규칙**: 적 관통 시 목숨 감소(잡몹 -1/보스 -10), **50라운드 최종 보스 관통 시 체력 무관 즉시 패배**
  + **시작 직전 경고**(토스트 + 미리보기 빨간 경고).
- **UI**: 모바일 가로 최적화. 핵심(건설/합성/가스/업그레이드/선택타워)은 도크에 상시, 부가(개인미션/테라진/퀘스트)는
  **☰ 드로어**. 다음 라운드 미리보기(잡몹수·HP), 보스/미션 HP 숫자 표시.

## 9. 배포 (GitHub Pages, 모바일 웹 플레이)
- **빌드는 GitHub Actions(Ubuntu)에서 수행** → 로컬 Node24+Windows 빌드 크래시와 무관.
- 설정 파일: `.github/workflows/deploy.yml`(Node20으로 빌드→Pages 배포), `vite.config.ts`의 `base: './'`.
- 절차:
  1. GitHub에서 새 **public** 리포 생성(README 등 초기화 없이).
  2. `git remote add origin https://github.com/<USER>/<REPO>.git` → `git push -u origin main`
  3. 리포 **Settings → Pages → Source = GitHub Actions** 로 설정.
  4. Actions 탭에서 배포 완료되면 `https://<USER>.github.io/<REPO>/` 접속(모바일 가능).
- 이후 `main`에 push할 때마다 자동 재배포.

## 8. 검증 방법(이번 세션에서 쓴 방식)
- 타입체크: `npx vue-tsc --noEmit`
- dev 서빙 확인: `npm run dev` 후 `http://localhost:5173/` 200 + 모듈 컴파일 에러 0
- 로직 헤드리스 테스트: `engine/`은 순수 TS라 esbuild로 번들해 node 실행 가능
  (예: `npx esbuild test.ts --bundle --platform=node --format=esm --outfile=t.mjs && node t.mjs`).
  GameEngine을 직접 import해 라운드/합성/미션/패배 등 시뮬레이션 검증.
