> 이번 세션의 모든 결정·진행 상황·미결 사항을 정리한 문서. (작성 시점 기준 최신)

## 0. 한 줄 요약
스타크래프트2 아케이드 **「랜덤 타워 디펜스」** 를 **Vue 3 + TypeScript(Vite)** 로 재구현. 가로 레이아웃,
Canvas 게임 필드 + Vue 반응형 HUD. 나무위키 구조 + 사용자 실측 데이터를 기준으로 밸런싱.

## 1. 다른 PC에서 시작하기
```bash
# 폴더 전체 복사(또는 git clone) 후
npm install
npm run dev      # http://localhost:5173 에서 플레이 (자동으로 브라우저 열림)
npx vue-tsc --noEmit   # 타입체크
```
- ⚠️ **프로덕션 빌드 주의**: `npm run build`는 **Node 24 + Windows** 조합에서 rollup 네이티브 단계
  크래시(`0xC0000409`)가 납니다(게임 코드 무관, hello-world도 동일). 빌드가 필요하면 **Node 20/22 LTS**
  에서 하세요. 개발/플레이는 `npm run dev`(esbuild)로 영향 없음. 자세한 건 [README.md](README.md).

## 4. 구현 완료 기능 (이번 세션 누적)
- **핵심 루프**: 50라운드·10마다 보스·50클리어 승리. 종족 미선택, **일반 타워만 랜덤 건설(100)**,
  **같은 종류 2개 합성 → 상위 등급**(일반→희귀→영웅→전설→신). 판매 반값(**확인 모달** 거침).
- **로스터 44종 = 뉴 랜타디 23 인게임 데이터**(일반8·희귀8·영웅8·전설8·신12). DPS/공속/사거리/종족 = 사용자 실측.
  - **공격 횟수(hits)**: 주기당 hits발 발사 → 진행도 상위 적들에게 분산(다중 타격). 발당=DPS×공속÷hits.
  - **다종족**: 메카 궤멸충[T,Z], 뫼비우스 혼종[T,P,Z] — 해당 종족 업글을 모두 합산.
  - **role(라인/보스/밸런스)·splash(범위) = 나무위키 서술 기준** 분류. splash 반경은 타워별 38~62px(임의 튜닝).
  - **근접 유닛**(사거리≤5: 저글링·광전사·용기병) = 근접 슬래시 모션 + 최소 사거리 62px 보장.
- **경제**: 시작 미네랄 300, 라운드 클리어 +300, 처치당 미네랄 없음(전부 50/100 단위). 가스 도박(100→20~120).
- **3종족 업그레이드**: 시작 20가스, 레벨당 +2가스, **공격력 +12.5%/레벨(단리)**, 최대 100업.
- **테라진**: 보스마다 +1, 750킬 +2, 3종족 신 +1 / 사용: 미네랄400·가스300·영웅 선택권(◆1+100원).
- **퀘스트 9종**(수집5 + 750킬 + 시간 7/12/18분), 보스/미션 클리어 + 달성 **토스트** 안내.
- **개인 미션 3단계**: 게/암흑광전사/대군주, 5분 쿨다운, 체력=나무위키 절대값(3000/30000/175000)×난이도,
  **roundBound 분리**(라운드 중 소환=종료 막음 / 대기 중 소환=라운드와 별개).
- **난이도 4종**: 쉬움/보통/어려움/지옥. **어려움=실제 랜타디 표준 체력(원게임 500%)**, 보통 80%·쉬움 60%,
  지옥은 어려움과 같은 체력 + 목숨 5 + 시작 보너스(미네랄200·가스100). (이름에 % 표기 없음)
- **체력**: `roundHp.ts` 50칸 실측 테이블(어려움/지옥=500% 기준), 난이도 `×hpMult/5`. 보스 라운드는 테이블값=보스HP.
- **잡몹 수**: 라운드당 **20마리 고정**, 보스 라운드 **1마리**.
- **이동속도**: 모든 몹 4.5(=108px/s) 공통.
- **무한모드**: 50클리어 후 테라진222 + 51R~ (체력 외삽 +15%/R).
- **패배 규칙**: 관통 시 목숨 감소(잡몹-1/보스-10), **50라운드 최종 보스 관통 = 체력 무관 즉시 패배** + 시작 직전 경고.
  체력 감소 시 **붉은 비네트 플래시 + '−N♥'**.
- **시간/진행**: 게임 클럭, 배속 1/2/3x·일시정지(직전 배속 기억, 시작 버튼으로도 해제),
  **첫 라운드만 수동, 이후 20초 카운트다운 후 자동 시작**(미리보기 칩 표시).
- **맵**: 사용자 스프레드시트 12×12 미로(576×576). 경로는 **벽에 막힐 때만 꺾이는 serpentine**, 셀 48px 스냅(셀당 1타워).
- **사거리**: 1유닛 = 0.5칸(`RANGE_SCALE=24`).
- **시각 효과**: 절차적 글리프(이모지 대체, iOS 안전), 등급 랭크별 발사체 글로우, **광역=은은한 링/단일=플래시/근접=베기**.
- **선택/정보**: 설치·합성 시 자동 정보 안 띄움. **직접 탭**해야 우상단 정보 표시. 설치/합성 모드에서 **이미 설치된 칸 탭** 시 정보.
- **UI(모바일 세로/가로)**: 전체화면 맵 + 오버레이.
  - 상단바: **R·시간·배속(좌) / 시작(우)**. 그 아래 스탯줄(미네랄·가스·체력·테라진·킬).
  - 하단바: **일반 타워·합성 모드·가스 도박·업그레이드·메뉴**. 업그레이드·메뉴는 **각 버튼 위로 펼치는 고정폭 팝오버**.
  - 선택 타워 정보 = 우상단 오버레이(글리프 아이콘). 줌 ＋－⊡ 버튼(하단바 위).
  - 더블탭 줌 차단(viewport + touch-action).

## 9. 배포 (GitHub Pages, 모바일 웹 플레이)
- **빌드는 GitHub Actions(Ubuntu)에서 수행** → 로컬 Node24+Windows 빌드 크래시와 무관.
- 설정 파일: `.github/workflows/deploy.yml`(Node20으로 빌드→Pages 배포), `vite.config.ts`의 `base: './'`.
- 절차:
  1. GitHub에서 새 **public** 리포 생성(README 등 초기화 없이).
  2. `git remote add origin https://github.com/<USER>/<REPO>.git` → `git push -u origin main`
  3. 리포 **Settings → Pages → Source = GitHub Actions** 로 설정(필수 — 워크플로에 자동 활성화 없음).
  4. Actions 탭에서 배포 완료되면 `https://<USER>.github.io/<REPO>/` 접속(모바일 가능).
- 이후 `main`에 push할 때마다 자동 재배포.
